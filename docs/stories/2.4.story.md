# Story 2.4: Dockerfile Parser

## Status

Done

## Story

**As an** operator,
**I want** to analyze Dockerfiles and extract base image references,
**so that** I know exactly which images need to be synchronized.

## Acceptance Criteria

1. `parse_dockerfile(path, build_args)` returns a list of `ResolvedImage` dataclasses
2. Handles pre-FROM ARGs with defaults and overrides
3. Missing ARG without default -> `ResolverError` with line info
4. Detects internal aliases (`is_internal: True`, skipped)
5. Handles `FROM scratch` -> skipped
6. Handles `--platform` flag -> extracts to `platform` field
7. Handles digests (`@sha256:...`) -> extracts to `digest` field
8. Handles `\` continuations and `#` comments
9. Docker Hub normalization (`nginx` -> `docker.io/library/nginx`)
10. Tests: multi-stage, ARGs, platform, digest, scratch, alias, continuation, errors

## Dev Notes

### ResolvedImage Data Model

```python
@dataclass
class ResolvedImage:
    raw: str              # Original FROM line value
    resolved: str         # Fully resolved reference
    registry: str         # e.g., "docker.io"
    repository: str       # e.g., "library/nginx"
    tag: str | None       # e.g., "1.25" or None if digest
    digest: str | None    # e.g., "sha256:abc..." or None if tag
    platform: str | None  # e.g., "linux/amd64" or None
    alias: str | None     # e.g., "builder" from AS builder
    is_internal: bool     # True if references earlier stage
    is_private: bool      # True if registry is in private list
    line_number: int      # Line number in Dockerfile
```

[Source: `docs/architecture/4-data-models.md#Resolved Dockerfile`]

### Docker Hub Normalization Rules

- `nginx` -> `docker.io/library/nginx:latest`
- `nginx:1.25` -> `docker.io/library/nginx:1.25`
- `myorg/myimage` -> `docker.io/myorg/myimage:latest`
- `ghcr.io/owner/repo:tag` -> kept as-is
- `nginx@sha256:abc...` -> `docker.io/library/nginx` with digest field

### Private Registry Detection

Private registries are defined in config (`private_registries` list). For this story, accept a list of patterns as parameter to `parse_dockerfile` or to the parser class. Patterns like `registry.internal`, `localhost:*`.

[Source: `docs/architecture/9-storage-schema.md` - config.json `private_registries` field]
