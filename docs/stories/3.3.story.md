# Story 3.3: Transfer - Generate Request & Import Response

## Status

Draft

## Story

**As an** operator,
**I want** to generate encrypted transfer requests and import encrypted responses,
**so that** I can securely sync layers via USB.

## Acceptance Criteria

1. `generate_request()` produces encrypted JSON with `version`, `buncker_version`, `source_id`, `blobs`
2. File named `buncker-request-{datetime}-{source_id}.json.enc`
3. `import_response()` decrypts -> HMAC verify -> SHA256 per blob -> atomic import -> cache manifests
4. `ERRORS.json` in response -> partial import, valid blobs kept
5. Logging: `transfer_manifest_generated` and `transfer_imported` events
6. Tests: round-trip, invalid HMAC rejected, corrupt blob rejected

## Tasks / Subtasks

- [ ] Task 1: Implement `generate_request()` (AC: 1, 2, 5)
  - [ ] Build request JSON: `{"version": "1", "buncker_version": "...", "generated_at": "ISO", "source_id": "...", "blobs": [...]}`
  - [ ] Serialize to bytes
  - [ ] Sign with `crypto.sign(data, hmac_key)` -> append signature
  - [ ] Encrypt with `crypto.encrypt(signed_data, aes_key)`
  - [ ] Write to `buncker-request-{datetime}-{source_id}.json.enc`
  - [ ] Log event `transfer_manifest_generated`
- [ ] Task 2: Implement `import_response()` (AC: 3, 4, 5)
  - [ ] Read `.tar.enc` file
  - [ ] Decrypt with `crypto.decrypt(data, aes_key)`
  - [ ] Verify HMAC signature - raise `TransferError` if invalid
  - [ ] Extract tar archive to temp directory
  - [ ] Read `ERRORS.json` if present - log warnings for failed blobs
  - [ ] For each blob in `blobs/sha256/`:
    - [ ] Verify SHA256 matches filename
    - [ ] Import via `store.import_blob()`
    - [ ] Skip corrupt blobs (log error, continue)
  - [ ] Cache manifests from response
  - [ ] Log event `transfer_imported` with counts
  - [ ] Return summary: imported count, skipped count, errors
- [ ] Task 3: Handle partial imports (AC: 4)
  - [ ] `ERRORS.json` format: `[{"digest": "...", "error": "..."}]`
  - [ ] Valid blobs are still imported
  - [ ] Summary includes both successful and failed counts
- [ ] Task 4: Write tests (AC: 6)
  - [ ] Test generate_request produces valid encrypted file
  - [ ] Test round-trip: generate -> (simulate fetch) -> import
  - [ ] Test import with invalid HMAC raises TransferError
  - [ ] Test import with corrupt blob: blob skipped, others imported
  - [ ] Test ERRORS.json handling
  - [ ] Test logging events are produced

## Dev Notes

### Transfer Request Format

```json
{
  "version": "1",
  "buncker_version": "0.1.0",
  "generated_at": "2026-03-04T10:00:00Z",
  "source_id": "buncker-prod-01",
  "blobs": [
    {"registry": "docker.io", "repository": "library/nginx", "digest": "sha256:abc...", "size": 12345, "media_type": "application/vnd.oci.image.layer.v1.tar+gzip"}
  ]
}
```

Wire format: `encrypt(json_bytes + HMAC_signature)`

[Source: `docs/architecture/4-data-models.md#Transfer Request`]

### Transfer Response Format (OCI Layout)

```
oci-layout
index.json
blobs/sha256/{digest_hex}
MANIFEST.sig          # HMAC of the entire tar before encryption
ERRORS.json           # Optional: list of blobs that failed to fetch
```

Wire format: `encrypt(tar_bytes + HMAC_signature)` -> `.tar.enc`

[Source: `docs/architecture/4-data-models.md#Transfer Response`]

### Crypto Operations

- Encrypt: `crypto.encrypt(data, aes_key)` -> AES-256-GCM
- Decrypt: `crypto.decrypt(data, aes_key)` -> raises CryptoError
- Sign: `crypto.sign(data, hmac_key)` -> HMAC-SHA256 hex
- Verify: `crypto.verify(data, hmac_key, signature)` -> bool

[Source: `docs/architecture/5-components.md#shared/crypto`]
