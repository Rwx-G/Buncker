# Story 4.1: Registry Client Online

## Status

Done

## Story

**As** buncker-fetch,
**I want** to authenticate and download blobs from any OCI registry,
**so that** I can fulfill transfer requests.

## Acceptance Criteria

1. Auth discovery: `GET /v2/` -> 401 -> parse `Www-Authenticate` -> extract realm, service, scope
2. Token exchange with Bearer token support and Basic Auth for credentials
3. `fetch_manifest()` returns manifest JSON
4. `fetch_blob()` returns blob bytes streaming
5. Credentials from `~/.buncker/config.json` via env vars
6. Retry 3x exponential backoff (1s, 3s, 9s). Timeout connect 30s, read 120s
7. HTTP 429 -> explicit message
8. Tests: mock HTTP server simulating Docker Hub auth

## Dev Notes

### OCI Auth Flow

```
1. GET /v2/ -> 401 + Www-Authenticate header -> parse realm/service/scope
2. GET {realm}?service={service}&scope={scope} -> Bearer token
3. GET /v2/{name}/manifests/{ref} + Authorization: Bearer {token}
4. GET /v2/{name}/blobs/{digest} + Authorization: Bearer {token}
```

[Source: `docs/architecture/6-external-apis.md`]

### Supported Registries

docker.io, ghcr.io, quay.io, gcr.io, any OCI-compliant custom registry.

**Docker Hub specifics:**
- Registry URL: `registry-1.docker.io`
- Auth realm: `https://auth.docker.io/token`
- Rate limit: 100 pulls/6h anonymous, 200/6h authenticated

[Source: `docs/architecture/6-external-apis.md`]

### HTTP Client

Use `urllib.request` (stdlib). HTTPS mandatory for all external requests. Certificate verification by default.

[Source: `docs/architecture/3-tech-stack.md`]

### Retry Policy

- 3 attempts, exponential backoff (1s, 3s, 9s)
- Connect timeout: 30s
- Read timeout: 120s
- HTTP 429: explicit message, respect Retry-After header if present

[Source: `docs/architecture/12-error-handling-strategy.md`, `docs/architecture/6-external-apis.md`]

### Credentials Security

- Registry credentials via env vars only, never plaintext in config
- Config maps registry names to env var names
- Online config: `~/.buncker/config.json`

[Source: `docs/architecture/15-security.md#Authentication`]

### Code Quality Assessment

`registry_client.py` is well-structured and readable. The `RegistryClient` class cleanly separates auth discovery, token exchange, request building, and response streaming. All public methods carry docstrings with Args/Returns/Raises. Constants for timeouts, retries, and backoff are module-level and clearly named. The `_stream()` helper buffers the full response before yielding chunks; a comment explains the design trade-off (avoid duplicate data on retry). The `load_credentials()` helper correctly reads from env vars and never logs credential values. The `_quote()` wrapper on `urllib.request.quote` is a minor convenience that could be inlined, but it causes no issues. Type hints are complete on all public and private signatures.

### Improvements Checklist

- [ ] Minor: `_stream()` loads the full blob into memory before yielding chunks. For very large blobs this negates streaming benefits. A future story could add true chunked streaming with a retry-on-partial-read guard. Not a blocker.
- [ ] Minor: Token expiry from the JWT `exp` field is not parsed; tokens are only evicted on a 401 re-challenge. Acceptable for MVP but noted for future improvement.
- [ ] Minor: `_quote` is a thin wrapper; could be inlined to reduce indirection. No functional impact.

### Security Review

- Credentials read exclusively from environment variables via `load_credentials()` - no plaintext in config. PASS
- HTTPS enforced for all registry calls (`https://` hardcoded in URL construction). PASS
- Certificate verification uses urllib default (system CA bundle). PASS
- No secrets logged: `_log.warning` for rate-limit and `_log.info` for retry log only status codes and timing, not tokens or credentials. PASS
- Basic auth header uses `base64.b64encode` in memory only, not persisted. PASS

### Performance Considerations

- Retry backoff (1s, 3s, 9s) is correctly implemented with `time.sleep`; `time.sleep` is patched in retry tests so tests run fast.
- Token caching per scope avoids redundant auth round-trips during a session.
- Blob streaming via `_stream()` currently buffers full blob in memory; acceptable for MVP layer sizes.
