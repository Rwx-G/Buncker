# Story 2.5: Resolver Pipeline

## Status

Draft

## Story

**As an** operator,
**I want** a complete analysis that tells me exactly which blobs are missing,
**so that** I can decide to generate a transfer request.

## Acceptance Criteria

1. `resolve_dockerfile(path, build_args, store, registry_client)` orchestrates: parse -> manifest lookup -> `list_missing`
2. Returns `AnalysisResult` with resolved images, present/missing blobs, total missing size
3. Deduplication: shared blob counted once
4. Private registries -> skipped with warning
5. Tag `latest` -> warning logged
6. Manifest not in cache -> image marked "unresolved" with explicit message
7. Tests: multi-image with shared blobs, private image skipped, missing manifest signaled

## Tasks / Subtasks

- [ ] Task 1: Define `AnalysisResult` dataclass (AC: 2)
  - [ ] Fields: `source_path`, `build_args`, `images` (list[ResolvedImage]), `present_blobs` (set), `missing_blobs` (list with registry, repository, digest, size, media_type), `total_missing_size`, `warnings` (list[str])
- [ ] Task 2: Implement `resolve_dockerfile()` orchestration (AC: 1, 3)
  - [ ] Call `parse_dockerfile()` to get image list
  - [ ] For each non-internal, non-scratch, non-private image:
    - [ ] Look up manifest in cache via `get_manifest()`
    - [ ] Extract layer digests from manifest
    - [ ] Call `store.list_missing()` with layer digests
  - [ ] Deduplicate: track seen digests across all images, count each blob once
  - [ ] Compute `total_missing_size` from missing blob sizes
- [ ] Task 3: Handle edge cases (AC: 4, 5, 6)
  - [ ] Private registry images: skip with warning "Private registry {registry} skipped"
  - [ ] Tag `latest`: add warning "Image {image} uses tag 'latest' - consider pinning"
  - [ ] Manifest not in cache: mark image as unresolved, add to warnings "Manifest not cached for {image} - run fetch first"
- [ ] Task 4: Write tests (AC: 7)
  - [ ] Test full pipeline: Dockerfile -> analysis with present + missing blobs
  - [ ] Test multi-image Dockerfile with shared layers (deduplication)
  - [ ] Test private image skipped with warning
  - [ ] Test unresolved manifest produces warning
  - [ ] Test latest tag produces warning
  - [ ] Test all-present scenario (no missing blobs)

## Dev Notes

### Resolver Pipeline Workflow

```
parse_dockerfile(path, build_args)
    |
    v
For each external image:
    get_manifest(registry, repo, tag) -- from cache
        |
        v
    Extract layer digests from manifest
        |
        v
    store.list_missing(layer_digests)
        |
        v
    Collect missing blobs (deduplicated)
```

[Source: `docs/architecture/7-core-workflows.md#Workflow 1`]

### AnalysisResult Data Model

```python
@dataclass
class AnalysisResult:
    source_path: str
    build_args: dict[str, str]
    images: list[ResolvedImage]
    present_blobs: set[str]       # digests already in store
    missing_blobs: list[dict]     # [{registry, repository, digest, size, media_type}]
    total_missing_size: int       # bytes
    warnings: list[str]
```

No specific architecture doc defines this dataclass - it's derived from the workflow and the PRD acceptance criteria.
