### Tech Stack (relevant to this story)

| Tool | Version | Purpose |
|------|---------|---------|
| Python | >=3.11 | Required minimum version |

| pytest | 8.x | Test framework |

[Source: `docs/architecture/3-tech-stack.md`]

### Coding Standards

- **Language:** Python >=3.11
- **Linting:** ruff with rules `E,F,W,I,UP,B,SIM`
- **Tests:** pytest, `tests/` directory mirrors source structure
- **No pip, no venv** constraint applies to production packages only; dev tools (ruff, pytest) are installed separately for development
- f-strings only, `pathlib.Path` for paths, type hints on public signatures

[Source: `docs/architecture/13-coding-standards.md`]

### CI Pipeline

CI order: `ruff check` -> `ruff format --check` -> `pytest (unit)` -> `pytest (integration)` -> `pytest (e2e)` -> `coverage`

For this story, only lint + unit test (placeholder) are needed. Integration and e2e stages will be added in later stories.

[Source: `docs/architecture/14-test-strategy.md`]

### Important Notes

- The `README.md`, `LICENSE`, `CONTRIBUTING.md`, `CHANGELOG.md`, `.gitignore`, and other root files already exist in the repo. This story only needs to add dev instructions to `README.md` and verify `.gitignore` completeness.
- The `.github/` directory already contains issue templates, CODEOWNERS, and PR template. The CI workflow file needs to be added.
- No existing code to build on - this is the first implementation story.

### Testing

- **Test location:** `tests/test_placeholder.py`
- **Framework:** pytest 8.x
- **This story only requires a single placeholder test** to verify CI works end-to-end
- The placeholder test should be simple (e.g., `assert True` or a basic import check)
- Future stories will add real tests following the `tests/` mirror structure
