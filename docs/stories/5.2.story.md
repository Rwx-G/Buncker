## Dev Notes

### E2E Test Architecture

E2E tests simulate the full USB cycle without real network. All external registries are replaced by a local mock HTTP server.

```
Mock Registry (localhost)
    ^
    |  (HTTP)
buncker-fetch (online side)
    |
    v (encrypted files)
buncker (offline side)
    |
    v (OCI API)
Docker pull simulation
```

[Source: `docs/architecture/14-test-strategy.md`]

### Test Pyramid

- 70% unit, 25% integration, 5% e2e
- E2E tests are the 5% - they cover critical paths, not every edge case
- Edge cases should already be covered by unit and integration tests

[Source: `docs/architecture/14-test-strategy.md`]

### CI Pipeline

Full pipeline: `ruff check` -> `ruff format --check` -> `pytest (unit)` -> `pytest (integration)` -> `pytest (e2e)` -> `coverage`

E2E tests must run without real network. Use mock HTTP servers on localhost.

[Source: `docs/architecture/14-test-strategy.md`]

### Improvements Checklist

- [ ] **E2E-001** (high): AC 5 states "All tests run in CI without real network". The e2e tests themselves use mock HTTP, but `ci.yml` does NOT run them. The test job runs `pytest` without `-m e2e` inclusion, and there is no separate e2e step. Add `pytest -m e2e` or simply ensure the default `pytest` run includes e2e tests (verify `addopts` does not exclude them).
- [ ] **E2E-002** (low): `conftest.py` uses `os.urandom(64)` for blob data at module import time. Add a brief comment explaining this is intentional for test isolation (different digests each run).
- [ ] **E2E-003** (low): `crypto_keys` fixture uses `iterations=1000` (vs 1,200,000 production). Add a comment: `# Reduced iterations for test speed`.
- [ ] **COV-001** (low): `pyproject.toml` does not enforce `--cov-fail-under=80` in `addopts`. Coverage threshold is only manually verified. Consider adding `--cov-fail-under=80` to catch regressions.

### Security Review

- Tests correctly validate: HMAC tampering rejected, wrong AES key rejected, corrupt blob SHA256 mismatch detected
- No secrets in test files - keys generated at runtime
- Mock server binds to `127.0.0.1` only - no external exposure

### Performance Considerations

- Session-scoped mock server: good, avoids per-test overhead
- `iterations=1000` for PBKDF2: appropriate for tests
- `parallelism=2` in fetcher tests: matches test scale
