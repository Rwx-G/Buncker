# Story 4.2: Fetcher & Cache

## Status

Draft

## Story

**As an** operator,
**I want** downloaded blobs cached locally and parallel downloads,
**so that** transfers are fast and resumable.

## Acceptance Criteria

1. Cache manages `~/.buncker/cache/blobs/sha256/`
2. `has_blob`/`get_blob`/`store_blob` with SHA256 verification
3. Fetcher downloads blob list with `ThreadPoolExecutor`
4. Cached blobs auto-skipped (resume)
5. SHA256 mismatch -> retry 3x -> errors list
6. Progress reporting
7. `cache clean --older-than Nd`
8. Tests: cache hit/miss, parallelism, corrupt blob retried

## Tasks / Subtasks

- [ ] Task 1: Implement Cache class (AC: 1, 2)
  - [ ] Create `buncker_fetch/cache.py`
  - [ ] `Cache(path)` initializes `~/.buncker/cache/blobs/sha256/` directory
  - [ ] `has_blob(digest)`: check file exists
  - [ ] `get_blob(digest)`: return Path to cached file
  - [ ] `store_blob(digest, data)`: atomic write + SHA256 verify + rename
  - [ ] Raise error on SHA256 mismatch after write
- [ ] Task 2: Implement Fetcher class (AC: 3, 4, 5, 6)
  - [ ] Create `buncker_fetch/fetcher.py`
  - [ ] Accept blob list and `RegistryClient` + `Cache`
  - [ ] Use `ThreadPoolExecutor` with configurable `parallelism` (default 4)
  - [ ] For each blob:
    - [ ] Check cache first (skip if present)
    - [ ] Fetch via `registry_client.fetch_blob()`
    - [ ] Verify SHA256
    - [ ] Store in cache
  - [ ] On SHA256 mismatch: retry up to 3 times
  - [ ] Collect errors for blobs that fail all retries
  - [ ] Return result: `FetchResult(downloaded, skipped, errors)`

  - [ ] Report: blob N/M, bytes downloaded, current blob name
  - [ ] Write to stderr (not stdout, which may be used for machine output)
  - [ ] Support `--json` mode (structured progress events)
- [ ] Task 4: Implement cache clean (AC: 7)
  - [ ] `cache_clean(older_than_days)` deletes blobs older than N days
  - [ ] Check file modification time
  - [ ] Return count and bytes freed
- [ ] Task 5: Write tests (AC: 8)
  - [ ] Test cache has_blob/get_blob/store_blob
  - [ ] Test cache store with SHA256 mismatch raises error
  - [ ] Test fetcher skips cached blobs
  - [ ] Test fetcher downloads missing blobs (mock registry)
  - [ ] Test parallel downloads work correctly

  - [ ] Test cache clean removes old files

## Dev Notes

### Cache Storage

```
~/.buncker/
  cache/blobs/sha256/{digest_hex}
```

Same atomic write pattern as the offline store: temp + verify SHA256 + rename.

[Source: `docs/architecture/9-storage-schema.md`]

### Concurrency

Use `concurrent.futures.ThreadPoolExecutor` - stdlib, simple, sufficient for I/O-bound downloads.

Default parallelism: 4 workers (configurable via `--parallelism` CLI flag).

[Source: `docs/architecture/3-tech-stack.md`]

### Expected Interfaces

The fetcher depends on these interfaces from Story 4.1:

```python
# buncker_fetch/registry_client.py (Story 4.1)
def fetch_blob(self, digest: str) -> Iterator[bytes]:
    """Fetch blob content as an iterator of byte chunks."""
```

The `FetchResult` return type:

```python
@dataclass
class FetchResult:
    downloaded: list[str]   # digests successfully fetched
    skipped: list[str]      # digests already in cache
    errors: list[dict]      # [{"digest": ..., "error": ...}]
```

### Critical Rules

- SHA256 verify on every blob write - no exceptions
- Atomic writes only

[Source: `docs/architecture/13-coding-standards.md`]
