# Story 3.2: Admin API

## Status

Draft

## Story

**As an** operator,
**I want** HTTP endpoints to manage the registry,
**so that** I can analyze Dockerfiles, generate transfers, and monitor the system.

## Acceptance Criteria

1. `POST /admin/analyze` accepts JSON, calls resolver, returns JSON report
2. `POST /admin/generate-manifest` generates `request.json.enc`. 409 if no analysis pending
3. `POST /admin/import` accepts `multipart/form-data`, executes verified import
4. `GET /admin/status` returns version, store stats, uptime
5. `GET /admin/gc/report` returns GC candidates
6. `POST /admin/gc/execute` accepts digests + operator
7. `GET /admin/logs` returns filtered logs
8. Input validation: digest regex, path traversal prevention
9. Integration tests for each endpoint

## Dev Notes

### Admin API Specification

| Method | Path | Request | Response |
|--------|------|---------|----------|
| POST | `/admin/analyze` | JSON `{dockerfile, build_args}` | JSON analysis report |
| POST | `/admin/generate-manifest` | (empty) | File download `.json.enc` |
| POST | `/admin/import` | `multipart/form-data` | JSON import summary |
| GET | `/admin/status` | - | JSON status |
| GET | `/admin/gc/report` | `?inactive_days=N` | JSON candidates |
| POST | `/admin/gc/execute` | JSON `{digests, operator}` | JSON result |
| GET | `/admin/logs` | `?event=X&since=ISO&limit=N` | JSON array |

No authentication in V1.

[Source: `docs/architecture/8-rest-api-spec.md`]

### Error Responses

Standard error format:
```json
{"error": "description", "code": "ERROR_CODE"}
```

HTTP status codes: 200 (success), 400 (bad input), 404 (not found), 409 (conflict), 500 (internal error).

> **Note:** This JSON error format and HTTP status code mapping are conventions defined by this story for the Admin API. The architecture doc (`docs/architecture/12-error-handling-strategy.md`) covers error handling strategy but does not prescribe this specific response format.

[Source: Story convention, inspired by `docs/architecture/12-error-handling-strategy.md`]
