# Story 5.1: .deb Packaging

## Status

Draft

## Story

**As a** release engineer,
**I want** both components packaged as .deb,
**so that** they can be installed on Debian/Ubuntu with proper dependency management.

## Acceptance Criteria

1. `make build-deb` produces `buncker` and `buncker-fetch` .deb files
2. `buncker.deb` installs: `/usr/bin/buncker`, `/usr/lib/buncker/`, `/etc/buncker/config.json`, `buncker.service`
3. `buncker-fetch.deb` installs: `/usr/bin/buncker-fetch`, `/usr/lib/buncker-fetch/`
4. Depends: `python3 (>= 3.11)`, `python3-cryptography`
5. `postinst` creates buncker user and directories
6. `dpkg -i` + `systemctl enable --now` = working daemon
7. `shared/` copied in each .deb
8. GitHub Actions builds .deb as artifacts

## Dev Notes

### Package Structure - buncker.deb

```
/usr/bin/buncker                    # Entry point wrapper
/usr/lib/buncker/
  buncker/                          # Package source
    __init__.py, __main__.py, server.py, handler.py, ...
  shared/                           # Shared modules (copy)
    __init__.py, crypto.py, oci.py, wordlist.py, ...
/etc/buncker/config.json            # Default config (conffiles)
/lib/systemd/system/buncker.service # systemd unit
```

### Package Structure - buncker-fetch.deb

```
/usr/bin/buncker-fetch              # Entry point wrapper
/usr/lib/buncker-fetch/
  buncker_fetch/                    # Package source
    __init__.py, __main__.py, fetcher.py, ...
  shared/                           # Shared modules (copy)
    __init__.py, crypto.py, oci.py, wordlist.py, ...
```

### Dependencies (debian/control)

```
Depends: python3 (>= 3.11), python3-cryptography (>= 41.0)
```

No pip, no venv. Everything installed via apt.

[Source: `docs/architecture/3-tech-stack.md`, `docs/architecture/11-infrastructure-and-deployment.md`]

### Deployment Strategy

Manual .deb installation (air-gap requirement). No apt repository.

CI builds .deb as artifacts on GitHub Actions.

[Source: `docs/architecture/11-infrastructure-and-deployment.md`]

### Rollback

`dpkg -i buncker_<previous_version>.deb`. Store persists across upgrades. Config protected via `conffiles`.

[Source: `docs/architecture/11-infrastructure-and-deployment.md#Rollback`]
