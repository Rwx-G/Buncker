# Story 2.1: Store Core

## Status

Done

## Story

**As an** operator,
**I want** a reliable blob store on the filesystem,
**so that** imported Docker layers are safely stored and retrievable.

## Acceptance Criteria

1. `Store(path)` initializes an OCI Image Layout (`oci-layout`, `index.json`, `blobs/sha256/`, `meta/sha256/`)
2. `import_blob(data, expected_digest)` writes atomically (temp + verify SHA256 + rename). Raises `StoreError` on mismatch
3. `has_blob(digest)` returns bool
4. `get_blob(digest)` returns file path. Raises `StoreError` if absent
5. `list_missing(digests)` returns list of absent digests
6. Each import creates/updates the sidecar metadata JSON
7. `update_metadata(digest, event)` updates `last_requested` and increments `request_count`
8. Unit tests with `tmp_path`: import, idempotent re-import, corrupt blob rejected, missing list

## Dev Notes

### Data Model - OCI Blob Metadata

Each blob has a JSON sidecar at `meta/sha256/{digest_hex}.json`:

```json
{
  "digest": "sha256:abc123...",
  "size": 12345,
  "media_type": "application/vnd.oci.image.layer.v1.tar+gzip",
  "image_refs": ["docker.io/library/nginx:1.25"],
  "first_imported": "2026-03-04T10:00:00Z",
  "last_requested": "2026-03-04T12:00:00Z",
  "request_count": 3,
  "gc_status": "active"
}
```

[Source: `docs/architecture/4-data-models.md#OCI Blob`]

### Storage Schema

```
/var/lib/buncker/
  oci-layout
  index.json
  blobs/sha256/{digest_hex}
  meta/sha256/{digest_hex}.json
  manifests/{registry}/{repo}/{tag}/{platform}.json
  logs/buncker.jsonl
```

For tests, use `tmp_path` as the store root.

[Source: `docs/architecture/9-storage-schema.md`]

### Critical Rules

- **Atomic writes only:** temp file + verify SHA256 + `os.rename`. Never corrupt the store.
- **SHA256 verify on every blob read/write:** No exceptions, no skip.
- **Idempotent:** Importing the same blob twice = noop (same digest = same file).

[Source: `docs/architecture/13-coding-standards.md`, `docs/architecture/12-error-handling-strategy.md`]
