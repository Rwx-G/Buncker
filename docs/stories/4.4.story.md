# Story 4.4: CLI buncker-fetch

## Status

Done

## Story

**As an** operator,
**I want** a complete CLI to manage the online side,
**so that** I can pair, inspect, fetch, and manage the cache.

## Acceptance Criteria

1. `buncker-fetch pair`: enter 12 words, derive keys, save config
2. `buncker-fetch inspect`: decrypt, display summary
3. `buncker-fetch fetch`: full cycle with `--output` and `--parallelism` options
4. `buncker-fetch status`: cache state
5. `buncker-fetch cache clean`: cleanup
6. Human-readable output by default, `--json` for machine output
7. All errors actionable with context
8. Tests: each subcommand tested

## Dev Notes

### CLI Commands Reference

| Command | Description |
|---------|-------------|
| `buncker-fetch pair` | Enter 12 words, derive keys, save config |
| `buncker-fetch inspect <file>` | Decrypt request, show summary |
| `buncker-fetch fetch <file> [--output PATH] [--parallelism N]` | Full cycle |
| `buncker-fetch status` | Cache stats |
| `buncker-fetch cache clean [--older-than Nd]` | Cleanup |

[Source: `docs/architecture/5-components.md#buncker-fetch`]

### Online Config Format

```json
{
  "derived_key_check": "encrypted_marker_base64",
  "salt": "base64...",
  "registries": {
    "docker.io": {"user_env": "BUNCKER_DOCKER_USER", "pass_env": "BUNCKER_DOCKER_PASS"},
    "ghcr.io": {"user_env": "BUNCKER_GHCR_USER", "pass_env": "BUNCKER_GHCR_PASS"}
  }
}
```

Credentials stored as env var references, never plaintext.

[Source: `docs/architecture/9-storage-schema.md`, `docs/architecture/15-security.md`]

### Pairing Workflow

```
Operator (offline): buncker setup -> displays 12 words
Human channel: operator writes words on paper
Operator (online): buncker-fetch pair -> enters 12 words -> keys derived
```

[Source: `docs/architecture/7-core-workflows.md#Workflow 5`]

### Error Handling

All errors must be actionable:
- "Pairing failed: word 'xyz' not in BIP-39 wordlist. Check spelling."
- "Decryption failed: wrong key or corrupted file. Re-pair if needed."
- "Registry error: HTTP 429 from docker.io. Rate limited, retry in 5 minutes."

[Source: `docs/architecture/12-error-handling-strategy.md`]
